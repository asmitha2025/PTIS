Frontend

Backend

The core backend lives in backend/ptis_core/. It is not a mock UI script: the tests, CLI, API, proof report, and frontend evidence all use the same core logic.

Bayesian engineUpdates destination probabilities from no-turn checkpoint observations.
Smart-link decision engineActivates only when confidence and receiving capacity are safe.
LWR/compliance modelModels two-wheeler density and compliance scaling with explicit units.
Field replay gateValidates real CSVs, rejects raw identifiers, and requires ground-truth coverage.

What Public Reviewers Should See

Give reviewers the map console, proof report, public evidence brief, and this guide. The public claim should be narrow, falsifiable, and reproducible.

Safe public wording: Validated on reproducible corridor scenarios, grounded with official Bengaluru planning references, audited against real Bengaluru CCTV detection data, and visualized on a real OSRM/OpenStreetMap route. Field replay is pending observed checkpoint and OD/ground-truth data.
